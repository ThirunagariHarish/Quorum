## Migration Steps

No database schema changes. No new required environment variables.

### Optional: expose `ANTHROPIC_API_KEY` check
If you want the new `/health/claude` endpoint to return a live latency figure rather than
`"not_configured"`, ensure `ANTHROPIC_API_KEY` is set in your `.env` / `.env.production` file.
The key was already required for agent operation; this is just surfaced more visibly now.

### Frontend dependency install
If running the frontend outside Docker, run:

`@codemirror/language`, `@codemirror/legacy-modes`, `@codemirror/state`, `@codemirror/view`,

## Known Issues

- **Tectonic not in base Docker image** â€” `Dockerfile.backend` is built from `python:3.12-slim`
  which does not ship Tectonic. The `/api/v1/papers/{id}/compile` endpoint will return
  `503 Service Unavailable` in production until Tectonic is added to the backend image.
  Track: add `RUN curl -fsSL https://drop.deno.land/tectonic-install.sh | sh` or install via
  `apt` in `Dockerfile.backend`.  Agent-side compilation (agents calling `LaTeXCompiler`
  directly) is unaffected â€” agents run in their own process context where Tectonic may
  already be present.
- **`/latex-editor` route requires auth** â€” the page redirects to `/login` if the session
  token is absent or expired. This is intentional.

## Rollback Plan

### Rollback procedure (Docker Compose)

If this release introduces a regression in production, roll back to `v0.1.0` by:

```bash
# On the VPS / deployment host
cd /opt/quorum

# 1. Identify the previous image tags (or git SHA)
docker images | grep quorum

# 2. Reset to the prior git commit
git fetch origin
git reset --hard <previous-sha>   # e.g. 24c0448

# 3. Rebuild and restart the affected services
docker compose -f docker-compose.prod.yml build --no-cache backend frontend
docker compose -f docker-compose.prod.yml up -d --no-deps backend frontend

# 4. Confirm health
curl -sf http://localhost:80/api/v1/health | python3 -m json.tool
```

**Previous SHA:** `24c0448` (`Fix setup endpoint to return access tokens after account creation`)

Infrastructure services (postgres, redis, minio) are unaffected â€” no schema changes were
made in this release, so no data rollback is needed.

### Rollback verification
After rolling back, confirm:
```bash
curl -s http://localhost:8000/api/v1/health          # should return {"status":"healthy"}
curl -s -o /dev/null -w "%{http_code}" \
  http://localhost:8000/api/v1/papers/test/compile -X POST
# should return 404 (route not registered on v0.1.0) â€” confirms rollback complete
```
